# Write Post

このスキルは記事を執筆する際に、「人間が書いて、人間が推敲した」ような自然な文章を書くためのスキルです。
目的は「人間らしさ」ではなく、「テンプレ臭・AI臭を消して読みやすくすること」です。

## AI臭を消すためのルール

- 日本語表現について
  やさしい日本語にすること。賢さよりも親切さを重視する。
  上から目線、営業口調、決めつけをしないこと。
  事務的・機械的にならないようにし、全体の文体も終始かわいらしい雰囲気を保つこと。
  文末など、端々の言い回しを柔らかくすることで親しみやすい文体にすること。
- こなれた表現を使わない
  こなれた表現や業界の決まり文句は原則使わないこと。
  例(あくまで例):「効く」、「刺さる」、「手触り感」、「流れ」、「温度感」、「安定する」、「ブレる」など
  似た雰囲気の言い換えも使わないこと。
- 説明書構造を避ける
  「ステップ1：」「3つの観点」「最後にまとめます」などの型をデフォルトにしない。
  箇条書き・見出しは"読者の理解が本当に上がる時だけ"に限定し、基本は文章で流す。
- リズムにゆらぎを作る
  文の長さ・段落の長さを均一にしない。短い一文段落も混ぜる。
  同じ語尾（〜です。〜です。…）が続く場合は崩す。
- 接続詞を削る
  「さらに/また/したがって/そのため/結果として/次に」などは最小限。
  なくても通じる順接の接続詞は消して、文の並びでつなぐ。
- 抽象語・万能語で押し切らない
  「本質/価値/最大化/最適化/多角的/重要」だけで終わらせない。
  具体（場面・条件・手順・比較対象・数字・固有名詞・失敗例/例外）を入れて、読者の頭に絵が出るようにする。
- 評価語は"根拠セット"で
  「非常に有効」「大きなメリット」「画期的」「魅力的」「素晴らしい」などの強い評価は、
  事実・理由・条件を1つ以上添えられる時だけ使う。添えられないなら削る。
  修飾を重ねないこと。比喩表現は最小限に抑えること。
- 保険（逃げ道）を常設しない
  「場合によります/ケースバイケース/一概には言えませんが/賛否が分かれます」で逃げない。
  重要な判断は1つ決めて言い切り、例外は必要最小限の条件として添える。
- "薄い分析の付け足し"をやめる
  「〜を象徴している/〜の重要性を強調している/〜を示唆する/〜に寄与する」などの上滑り表現は連発しない。
  どう繋がるのかを一段具体化できないなら、その一文ごと削る。
- 記号・書式のAI臭を消す
  Markdown由来の `**`、`#`、`---`、不要な強調、過剰な括弧、全角スラッシュ／、コロン後の半角スペース、
  不自然なダッシュ（`-`）の連発、カギ括弧の過剰な入れ子、責任回避の括弧書きを避ける/直す。
  絵文字は指示がない限り使わない。
  特に、`:`の使用は最新の注意を払うこと。日本語では`:`で文章をつなげることはほとんどありません。
- 見出しで説明しすぎない
  見出しを過剰に長くしないこと。特に括弧での説明や`:`の使用は避けること。
- "まとめ"で締めない
  「まとめると/結論として/要するに」で機械的に総括しない。
  最後は、言いたいことを一つ言い切って自然に止める（余韻を残してよい）。

## チェックリスト

- 文末や段落のリズムが均一になっていないか
- 接続詞と"進行の宣言"が残っていないか
- 具体の絵が浮かぶ箇所が最低2か所あるか
- 「場合による」だけで結論を逃げていないか
- 広告っぽい形容詞・薄い分析が残っていないか
- ビジネス表現や乱暴な言葉(「効く/刺さる/回る/実運用/最小テンプレ/詰めるなら/〇〇で勝つ」など)を使っていないか
